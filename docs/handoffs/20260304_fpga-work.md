### Context

- **Phase 1 done:** `fpga-build.sh`, `fpga-deploy.sh`, `server-build-*.sh`, `server-deploy.sh`, `systemd-service-deploy.sh` exist and are adapted for rp_ads1278.
- **Strategy:** Use **RedPitaya axi4lite** as base. Do **not** use rpll `system.tcl` — it is too complex.
- **Integration plan (high confidence):**
  1. Add RTL: `ads1278_spi_tdm.v`, `ads1278_extclk_gen.v`, `ads1278_sync_pulse.v`, `ads1278_acq_top.v`, `ads1278_axi_slave.sv`
  2. In `red_pitaya_top.sv`, replace `axi4lite_gpio` with `ads1278_axi_slave` (which instantiates `ads1278_acq_top`)
  3. Connect expansion pins directly to `ads1278_acq_top` (SCLK, MISO, DRDY, SYNC, EXTCLK) — no IOBUF/gpio indirection
  4. Leave `systemZ10.tcl` unchanged (M_AXI_GP0 port already correct)
- **ADS1278 SPI** (`ADS1278_SPI.md`): CPOL=0, CPHA=0; sample MISO on SCLK **rising** edge; 192 SCLKs per frame (8×24 bits, CH1..CH8, MSB first); wait ≥1 CLK period after DRDY falls before first SCLK.
- **AXI slave pattern** (`axi4lite_gpio.sv`): Use `axi4_lite_if.s`; same handshake (AWREADY, WREADY, BVALID, ARREADY, RVALID); extend address decode for CH1..CH8 (read-only), STATUS (read-only), CTRL (R/W), EXTCLK_DIV (R/W).
- **Constraints:** `.reference/` is read-only; copy/adapt into `fpga/`. `README.md` is the source of truth.
- **Gaps:**
  - **E1 pin mapping (Mistake identified):** The `README.md` previously specified E2 pins for SPI. However, Red Pitaya E2 SPI pins are Zynq PS MIO pins, which cannot be directly driven by the FPGA PL logic. We MUST use the E1 expansion connector (`exp_p_io` / `exp_n_io`) for all connections (SCLK, MISO, DRDY, SYNC, EXTCLK).
  - **Shared RTL paths:** axi4lite uses `../../rtl` (e.g. `axi4_lite_if.sv`, `red_pitaya_pll`, `gpio_if`). Confirm which files rp_ads1278 needs when adapting the build.
